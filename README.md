<div style="background: #1e2338; border-radius: 24px; padding: 1.5rem; margin: 25px 0;">
    <h2 style="color: #FFD93D; margin-top: 0;">🧠 How It Works</h2>
    <p><strong>Collaborative Filtering using Cosine Similarity</strong></p>
    <ol style="line-height: 1.7;">
        <li>User–book rating matrix created (706 books × 810 power users)</li>
        <li>Empty ratings filled with 0</li>
        <li>Cosine similarity computed between all book vectors</li>
        <li>For a selected book, system returns top N most similar books</li>
        <li>Similarity score (0–100%) shows match percentage</li>
    </ol>
    <p><strong>Popularity Filter</strong> – Books sorted by rating count, filtered by threshold (≥250 ratings), then by average rating.</p>
</div>

<!-- CHALLENGES SOLVED -->
<div style="background: #1e2338; border-radius: 24px; padding: 1.5rem; margin: 25px 0;">
    <h2 style="color: #FFD93D; margin-top: 0;">⚡ Challenges & Solutions</h2>
    <ul>
        <li><strong>Large dataset (1.1M+ ratings)</strong> – Used <code>low_memory=False</code> and filtered to power users (≥200 ratings)</li>
        <li><strong>Sparse rating matrix</strong> – Filled NaN with 0 for cosine similarity computation</li>
        <li><strong>Memory constraints</strong> – Precomputed similarity matrix (706×706) saved as pickle</li>
        <li><strong>Missing cover images</strong> – Added fallback placeholder image on error</li>
        <li><strong>Streamlit re‑runs</strong> – Used session state to persist recommendations</li>
    </ul>
